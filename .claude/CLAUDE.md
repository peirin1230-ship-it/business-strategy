# CLAUDE.md — 病院経営分析ルール

東邦大学医療センター大橋病院の経営分析における作業ルール。

## 対象病院

- **大橋病院**（320床）: 主分析対象。DPC標準病院群 / 急性期一般入院料1 / 急性期充実体制加算 / 地域医療支援病院
- **大森病院**（800床）・**佐倉病院**（380床）: グループ内ベンチマーク比較対象

## データの取り扱い

### 出典の明記
- すべての数値に出典（資料番号）を付記する
- 同じ指標が複数資料に存在する場合（例: 入院単価が資料1と資料1-1で異なる）、1つのテーブル内では同一資料で統一する
- 資料間で数値が異なる場合は注記する（例: ICU算定率 資料1-1: 79.7% vs 資料4: 79.6%）

### 期間の一致
- 前年度対比は必ず**同一期間**で比較する（4-2月累計 vs 前年4-2月累計）
- 当年度4-2月（11ヶ月）と前年度通年（12ヶ月）を比較してはならない
- やむを得ず異なる期間を比較する場合は、明示的に注記する

### 事実と解釈の分離
- **【事実】**: 資料から直接読み取れる数値と傾向。出典を明記
- **【解釈】**: 事実に基づく推察・仮説。推論であることを明示
- 事実と解釈を混在させない

### 計算の正確性
- 増減率の計算: (当年値 - 前年値) / 前年値 × 100
- 差分と増減率は手計算で検証してから記載する
- 金額の単位（千円、万円、億円）の変換に注意する

## 成果物の構成

### 4つの成果物
1. **経営分析レポート**（report/）: 定量分析。予算対比と前年度対比の両面で評価
2. **幹部説明原稿**（presentation/）: 資料番号順。簡潔・事実中心・口頭説明用
3. **想定Q&A集**（qa/）: 【結論】【根拠データ】【補足/対策】の3段構成
4. **インサイトレポート**（insights/）: 5視点統合分析フレームワーク

### インサイトレポートの5視点
| 記号 | 視点 | 内容 |
|---|---|---|
| [データ] | 数値事実 | 出典付きの定量データ |
| [臨床] | 臨床現場 | 医師・看護師・コメディカルへの影響 |
| [負荷] | 現場負荷 | ★/★★/★★★で評価、緩和策を付記 |
| [請求] | 保険請求 | 算定コード・要件・査定リスク・算定漏れ |
| [経営] | 経営アクション | 収益インパクトの定量化と実行計画 |

## 分析の重点

### 予算対比と前年度対比
- 予算対比だけでなく、前年度実績との比較を必ず併記する
- 予算未達でも前年度比で改善している場合はその旨を明記する

### 3病院比較の深掘り
- 大橋病院が3病院中で**劣位にある指標**を重点的に分析する
- 劣位指標には★マークを付与し、原因と改善策を記載する

### 同クラス病院との外部ベンチマーク
- 急性期一般入院料1算定病院（n=270、平均271床）のデータで比較
- 出典: WAM「病院の経営状況等について」、厚労省「病院報告」

### 診療科別の分析
- 資料1-2に基づき、成長科と縮小科を分類する
- 外来と入院を合算した総合評価で判断する
- 増減の主な費目（注射・手術・手術材料等）を特定し、臨床的な要因を推察する

### 現場負荷への配慮
- すべての改善提言に、看護師・コメディカルの業務負担への影響を評価する
- 「追加業務」ではなく「既存業務への組み込み」を優先する

## 品質保証

### データ整合性チェック
- 成果物完成後、全数値を原本資料と突合する
- 特に以下を重点チェック:
  - 増減率の計算結果
  - 期間の一致（4-2月 vs 通年の混同がないか）
  - 資料間の数値の整合性（資料1 vs 資料1-1等）
  - 合計値と内訳の整合

### 会議メモの取り扱い
- 会議での口頭発言と資料データに齟齬がある場合は、**資料データを優先**する
- 会議固有の情報（方針決定、議論内容等）はそのまま記録する

## 月次運用

1. `data/YYYYMM/raw/` に戦略会議資料PDFを格納
2. PDFからデータを抽出し `data/YYYYMM/extracted/` に保存
3. 4つの成果物を作成し `analysis/YYYYMM/` 配下に格納
4. データ整合性チェックを実施
5. `python generate_pdf.py` でPDF出力
6. 会議メモがある場合は `analysis/YYYYMM/meeting/` に格納

## ファイル命名規則

- 成果物: `{種類}_YYYYMM.md` / `.pdf`（例: `経営分析レポート_202602.md`）
- 抽出データ: `（資料X）{資料名}_抽出結果.md`
- 会議メモ: `戦略会議議事要約_YYYYMMDD.md`
